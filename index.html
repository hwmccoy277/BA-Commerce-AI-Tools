Getting Started

Install Guide

This connects Triple Whale to Claude Desktop so you can have real conversations about your store data. Setup takes about 10 minutes.

What you'll need

🖥️
Claude Desktop
The desktop app for Claude. Download it here — it's free.
🔑
Triple Whale API Key
Found in your Triple Whale account under Settings → API.
⚙️
Node.js
A free tool that runs the server. Download the LTS version.
🏪
Your Shopify Store URL
In the format yourstore.myshopify.com — find it in Shopify admin.

Step-by-step setup

  1. 1

    Install Node.js

    Go to nodejs.org and download the LTS version. Run the installer — no special options needed. Once done, you won't need to touch it again.

  2. 2

    Download the MCP server

    If you've been given a ZIP file, unzip it somewhere easy to find (like your Desktop or Documents). If you're using git, run:

    Terminal 
    git clone https://github.com/your-org/tw-mcp.git
cd tw-mcp
  3. 3

    Build the server

    Open Terminal (Mac) or Command Prompt (Windows), navigate to the folder you just downloaded, and run these two commands:

    Terminal 
    npm install
npm run build

    You should see a dist/ folder appear when it's done. If you see errors, check that Node.js installed correctly.

  4. 4

    Open the Claude Desktop config file

    This is the file that tells Claude which tools to load. Open it in any text editor (Notepad, TextEdit, VS Code).

    File location:
    Mac:
    Path 
    ~/Library/Application Support/Claude/claude_desktop_config.json
    Windows:
    Path 
    %APPDATA%\Claude\claude_desktop_config.json
    💡 On Mac, the Library folder is hidden by default. In Finder, press Cmd + Shift + G and paste the path above to navigate there directly.
  5. 5

    Add Triple Whale to the config

    Inside the config file, find the "mcpServers" section and add the following block. Replace the placeholders with your actual values.

    claude_desktop_config.json 
    "triple-whale": {
  "command": "node",
  "args": ["/path/to/tw-mcp/dist/index.js"],
  "env": {
    "TRIPLE_WHALE_API_KEY": "your-api-key-here",
    "TRIPLE_WHALE_SHOPS": "yourstore.myshopify.com:Your Store Name"
  }
}
    📁 Replace /path/to/tw-mcp with the actual folder path where you downloaded the server. On Mac this might be something like /Users/yourname/Documents/tw-mcp.
  6. 6

    Set up your stores

    The TRIPLE_WHALE_SHOPS value lists every store you want to access. Format is shopify-url:Display Name, with multiple stores separated by commas:

    Multiple stores example 
    "TRIPLE_WHALE_SHOPS": "store-one.myshopify.com:Brand One,store-two.myshopify.com:Brand Two"
  7. 7

    Restart Claude Desktop

    Fully quit Claude Desktop (don't just close the window — right-click the icon and choose Quit) and reopen it. This is important — it won't load the new config until you do.

  8. 8

    Test it out

    Open a new chat in Claude and ask:

    Test prompt 
    What shops do I have configured?

    If everything worked, Claude will list your stores. If you see an error, double-check the file path in step 5 and make sure you fully restarted Claude.

How To Use It

Best Practices

You don't need special commands or technical knowledge. Just talk to Claude like you'd talk to an analyst. Here's how to get the most out of it.

Vague vs specific — the difference it makes

❌ Too vague ✅ Much better
"How are ads doing?" "How are Facebook and Google ads performing for Sunny Within this month? Show me ROAS and CPA."
"Show me customers" "What's the new vs returning customer split for The Fidget Games in the last 30 days?"
"Check creatives" "Are any of our TikTok creatives showing fatigue for FOCL in the last 60 days?"
"How's revenue?" "How did Sophie Grace's revenue compare this month vs last month?"
"Check Facebook" "Show me which Facebook ads are driving the most new customer revenue for [store] this month, sorted by NC ROAS."
"How's the website?" "Show me the conversion funnel for [store] this month — where are visitors dropping off between landing and purchase?"

7 tips that make a real difference

  • 1
    Always name the store. If you have multiple stores set up, Claude needs to know which one you mean. Use the display name you set up (e.g. "Sunny Within" or "FOCL").
  • 2
    Always include a time period. "This month", "last 30 days", "Q1", "year to date" — the more specific, the better. Without one, Claude will default to the last 30 days.
  • 3
    Start broad, then drill down. Ask for a business summary first. See something interesting? Then ask: "Can you break that down by channel?" or "Which campaigns are driving that?" Claude remembers the whole conversation.
  • 4
    Compare periods whenever possible. A number on its own tells you less than a number vs last month or vs last year. Ask for comparisons regularly — they surface problems and wins you'd otherwise miss.
  • 5
    Trust Triple Whale's ROAS over platform ROAS. Facebook and Google both claim credit for more sales than they actually drove. Triple Whale's pixel tracks the full picture. If the numbers differ, the TW number is more accurate.
  • 6
    Check for creative fatigue weekly. Ads naturally wear out — people stop clicking after seeing them too many times. Make creative fatigue a standing part of your weekly review before you decide on budget changes.
  • 7
    Use alerts as your Monday morning check-in. Ask "are any channels losing money this week?" at the start of each week. It takes 10 seconds and can save you from letting wasted spend run all week unnoticed.
  • 8
    Use NC ROAS to find your real growth ads. Overall ROAS includes returning customers who would have bought anyway. NC ROAS tells you which ads are actually growing your customer base. An ad with a great overall ROAS but terrible NC ROAS is mostly just retargeting people who already know you — not building the business.
